#openclaw

### SOUL.md

- 定义：
	- 性格
	- 语气
	- 价值观
	- 行为风格
	- 互动边界
- 回答的是：
	- **“你是谁？”**
- 这个最像灵魂设定。 

### AGENTS.md

- 这个名字虽然容易和“agent 实例”混，但它很重要。 它定义的是：
	- 工作原则
	- 读哪些文件
	- 安全规则
	- 群聊规则
	- 记忆维护规则
	- 外部/内部动作边界
	- heartbeat 行为
	- destructive 操作规范
- 回答的是：
	- “你怎么工作？”
- 不是人格，而是**操作系统规则**。 如果少了它，agent 很容易性格鲜明但做事没章法。 


### USER.md

- 定义：
	- 服务对象是谁
	- 怎么称呼
	- 用户偏好
	- 对方的背景、目标、脆弱点、习惯
- 回答的是：
	- “你是为谁而存在的？”
- 这个很关键。 同样一个 SOUL，面对不同用户，工作方式会完全不同。 

### IDENTITY.md

- 这个更像 SOUL 的结构化摘要。 定义：
	- 名字
	- creature / 角色
	- vibe
	- emoji
	- 核心身份描述
- 回答的是：
	- “你在系统里叫什么、以什么身份出现？”
- SOUL 更像散文，IDENTITY 更像身份证。 

### MEMORY.md

- 定义长期稳定记忆：
	- 用户偏好
	- 重要规则
	- 关键项目背景
	- 历史教训
	- 长期有效的判断
- 回答的是：
	- “你应该长期记住什么？”
- 这个不是设定文件，但它实际上会深刻塑造 agent。 

### HEARTBEAT.md

- 定义主动行为：
	- 心跳时要检查什么
	- 什么时候安静
	- 什么时候提醒
	- 周期任务怎么跑
- 回答的是：
	- “你不被点名时，平时该怎么活着？”
- 如果这个 agent 有主动性，这份文档很关键。 

### TOOLS.md

- 定义本地工具偏好和环境细节：
	- 本地路径
	- 常用脚本
	- 设备名字
	- TTS 偏好
	- SSH / camera / 环境说明
- 回答的是：
	- “你手里有哪些顺手家伙？”
- 这个不是人格，但很影响实战效率。 

---

## 哪些是“定义一个 agent”的核心文档？

### 最小可用集

- **SOUL.md** — 我是谁
- **AGENTS.md** — 我怎么工作
- **USER.md** — 我服务谁
- **MEMORY.md** — 我记住什么

这四个已经能把一个 agent 立起来了。

---


## 哪些是“高质量 agent”的完整集合？


- **IDENTITY.md** — 身份摘要
- **SOUL.md** — 灵魂/人格
- **AGENTS.md** — 行为与安全规则
- **USER.md** — 用户画像
- **MEMORY.md** — 长期记忆
- **HEARTBEAT.md** — 主动行为规范
- **TOOLS.md** — 环境与工具笔记
- **ROUTING.md**（可选）— 任务分发/多 agent 协作规则

---

## 推荐的设计思路

不是“一个总说明书”，而是：

### 人格层

- SOUL.md
- IDENTITY.md

### 规则层

- AGENTS.md
- HEARTBEAT.md
- ROUTING.md（可选）

### 对象层

- USER.md

### 连续性层

- MEMORY.md

### 环境层

- TOOLS.md


---

### 多实例系统

#### 每个实例都应该有自己独立的

- SOUL.md
- AGENTS.md
- USER.md
- MEMORY.md
- TOOLS.md

#### 可以部分共享模板的

- HEARTBEAT.md
- ROUTING.md
- 某些安全规则
- 某些基础工作规范

也就是说：**人格要独立**
- 记忆要独立
- 规则可以继承再微调
