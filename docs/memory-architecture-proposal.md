## 一、当前问题

现有 MEMORY.md 是个空壳，缺乏结构：
- 所有东西往一个文件塞，没有分层
- 没有反思机制，只记事实不回顾判断
- 没有检索优先级，启动时一股脑全读
- 私有记忆和共享记忆的边界模糊

---

## 二、三层记忆架构

```
每个 Agent 目录/
├── MEMORY.md              ← 长期稳定认知（提炼后的结论）
├── reflections/           ← 每日反思日志（原始观察）
│   ├── 2026-03-10.md
│   ├── 2026-03-11.md
│   └── ...
│
共享层（Obsidian Vault）/
├── 品牌规范/              ← 事实 + 规范（不可变，除非亚臣确认）
├── 产品信息/
├── 竞品档案/              ← Radar 可写
├── 战略决策记录/           ← Partner 可写
├── 交付物存档/             ← Workshop 可写
└── 审核规则库/             ← Keeper 可写
```

## 三、三层记忆的边界

| | MEMORY.md（私有） | reflections/（私有） | 共享层（Obsidian） |
|---|---|---|---|
| **存什么** | 提炼后的结论 | 当天的原始观察 | 事实、规范、档案 |
| **性质** | 经验、偏好、规则 | 过程、疑问、反思 | 团队共识 |
| **谁写** | 自己 | 自己 | 按权限 |
| **谁读** | 只有自己 | 只有自己 | 所有工作 agent |
| **更新频率** | 每周从 reflections 提炼 | 每天 | 事件驱动 |
| **类比** | 你的原则笔记 | 你的日记 | 公司知识库 |

---

## 四、每个 Agent 的 MEMORY.md 存什么

| Agent | 长期稳定认知 | 不存什么 |
|-------|------------|---------|
| **Radar** | 信息源质量评估、亚臣的关注领域权重、过滤经验 | 具体的新闻内容（那是共享层竞品档案的事） |
| **Partner** | 亚臣的决策风格、思维偏好、反复出现的关注点 | 具体的战略结论（那是共享层战略决策记录的事） |
| **Workshop** | 生产标准、brief 常见歧义、亚臣的交付偏好 | 具体的交付物内容（那是共享层交付物存档的事） |
| **Keeper** | 审核标准演化、高频问题类型、被推翻的否决反思 | 具体的法规条文（那是共享层审核规则库的事） |
| **Butler** | 生活习惯、偏好、重要日期、分类规则 | 具体的财务数额（不落盘） |

## 五、每日反思机制

每个 agent 在当天结束（或 heartbeat 触发时）写 `reflections/YYYY-MM-DD.md`：

```markdown
# YYYY-MM-DD 反思

## 今天的判断回顾
- [做对了什么/做偏了什么]

## 规则需要调整吗
- [哪条规则太松/太紧]

## 偏好更新
- [亚臣表达了什么新的偏好]

## 值得沉淀的经验
- [什么值得提炼进 MEMORY.md]
```

### 流转机制

```
每天 → reflections/YYYY-MM-DD.md（原始记录）
每周 → 回顾本周 reflections，提炼有长期价值的 → 写入 MEMORY.md
每月 → 回顾 MEMORY.md，清理过时条目
```

## 六、检索优先级

每个 agent 启动或响应前，按以下顺序检索（不是全读，是按需读）：

```
第1层：MEMORY.md（自有长期记忆）
       → 最先读。这是"你是谁"的记忆延伸

第2层：共享知识库（与当前任务相关的部分）
       → 按需读。不是全读，只读与当前上下文相关的

第3层：近期 reflections（最近 3-7 天）
       → 可选读。当需要回顾近期经验时

第4层：外部输入（Radar 简报、Brief、交付物等）
       → 被动接收。不主动检索
```

### 各 Agent 的检索侧重

| Agent | 第1层重点 | 第2层重点 |
|-------|----------|----------|
| Radar | 信息源清单、过滤经验 | 竞品档案（检查重复） |
| Partner | 亚臣偏好、历史决策 | 品牌规范、产品信息（对话素材） |
| Workshop | 生产标准、交付偏好 | 品牌规范、产品信息（执行依据） |
| Keeper | 审核标准、高频问题 | 法规规则库、品牌规范（审核依据） |
| Butler | 习惯、偏好、重要日期 | 不读共享层 |
